# LoComm - Publishing & Deployment Guide

This guide walks you through publishing LoComm to GitHub and setting up Supabase for app updates.

## Prerequisites

- Git installed: https://git-scm.com/
- GitHub account: https://github.com/
- Supabase account: https://supabase.com/

---

## Part 1: GitHub Repository Setup

### 1. Install Git

Download from: https://git-scm.com/

### 2. Create GitHub Repository

1. Go to https://github.com/new
2. Repository name: `locomm`
3. Description: "Local Voice Communication - PTT voice chat for local networks"
4. Public
5. Click "Create repository"

### 3. Initialize Local Repository

```bash
cd LoComm
git init
git add .
git commit -m "Initial release v1.0.0"
```

### 4. Connect and Push

```bash
git branch -M main
git remote add origin https://github.com/dev-j33zy/locomm.git
git push -u origin main
```

---

## Part 2: Supabase Setup for Updates

### 1. Create Supabase Project

1. Go to https://supabase.com/
2. New project
3. Name: locomm-updates
4. Database password: (save this)

### 2. Create Updates Table

Run this SQL in Supabase SQL Editor:

```sql
-- Create updates table
CREATE TABLE updates (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  app_name TEXT NOT NULL,
  platform TEXT NOT NULL,
  version TEXT NOT NULL,
  download_url TEXT NOT NULL,
  release_notes TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable row-level security
ALTER TABLE updates ENABLE ROW LEVEL SECURITY;

-- Allow public read access
CREATE POLICY "Public updates read" ON updates
  FOR SELECT USING (true);

-- Create index
CREATE INDEX idx_updates_app_platform ON updates(app_name, platform);
```

### 3. Add Your First Update

Insert your GitHub release URL:

```sql
INSERT INTO updates (app_name, platform, version, download_url, release_notes)
VALUES (
  'locomm',
  'web',
  '1.0.0',
  'https://github.com/YOUR_USERNAME/locomm/releases',
  'Initial release'
);
```

### 4. Get Supabase Credentials

1. Go to Project Settings > API
2. Copy Project URL
3. Copy anon public key

### 5. Configure Frontend

Create `.env.local` in `frontend/`:

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_KEY=your-anon-key
```

Rebuild frontend:
```bash
cd frontend
npm run build
```

## Part 3: Deployment Options

### Option A: Vercel (Recommended)

1. Go to https://vercel.com/
2. Import GitHub repository
3. Set build command: empty
4. Output directory: backend (select)
5. Deploy

### Option B: Netlify

1. Go to https://netlify.com/
2. Drag & drop `backend` folder
3. Note the URL

### Option C: Render

1. Go to https://render.com/
2. Create Web Service
3. Connect GitHub repo
4. Build command: npm start

## Part 4: GitHub Releases

### 1. Create Release

1. Go to your GitHub repository
2. Click "Create a new release"
3. Tag version: v1.0.0
4. Release title: v1.0.0
5. Upload files (backend/dist)
6. Publish release

### 2. Update Supabase

```sql
INSERT INTO updates (app_name, platform, version, download_url, release_notes)
VALUES (
  'locomm',
  'web',
  '1.0.1',
  'https://github.com/YOUR_USERNAME/locomm/releases/tag/v1.0.1',
  'Bug fixes and improvements'
);
```
