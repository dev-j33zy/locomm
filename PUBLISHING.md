# LoComm — Publishing & Release Guide

This guide covers GitHub publishing, optional Supabase update-check integration, and Vercel frontend deployment.

---

## Part 1: GitHub Repository Setup

### 1. Install Git

Download from: https://git-scm.com/

### 2. Create a GitHub Repository

1. Go to https://github.com/new
2. Repository name: `locomm`
3. Description: `Real-time PTT voice communication for local networks`
4. Visibility: **Public** (or Private)
5. Click **Create repository**

### 3. Initialize and Push

```bash
cd LoComm

git init
git add .
git commit -m "feat: initial release v1.0.0"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/locomm.git
git push -u origin main
```

### 4. Set Up .gitignore

The project includes a root `.gitignore` that excludes:
- `node_modules/` (both frontend and backend)
- `frontend/dist/` (build output)
- `.env` and `.env.local` (secrets)
- `backend/key.pem` and `backend/cert.pem` (SSL certificates)
- `backend/supabase-pass.txt`

## Part 2: Supabase Update Checker (Optional)

The frontend optionally integrates with Supabase to display an in-app update banner when a new version is available.

### 1. Create a Supabase Project

1. Go to https://supabase.com/
2. New project → name: `locomm-updates`
3. Save your database password

### 2. Create the Updates Table

Run this SQL in the Supabase SQL Editor:

```sql
CREATE TABLE updates (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  app_name TEXT NOT NULL,
  platform TEXT NOT NULL,
  version TEXT NOT NULL,
  download_url TEXT NOT NULL,
  release_notes TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

ALTER TABLE updates ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public updates read" ON updates
  FOR SELECT USING (true);

CREATE INDEX idx_updates_app_platform ON updates(app_name, platform);
```

### 3. Insert Your First Version

```sql
INSERT INTO updates (app_name, platform, version, download_url, release_notes)
VALUES (
  'locomm',
  'web',
  '1.0.0',
  'https://github.com/YOUR_USERNAME/locomm/releases',
  'Initial release'
);
```

### 4. Get Supabase Credentials

1. Go to **Project Settings → API**
2. Copy the **Project URL**
3. Copy the **anon/public key**

### 5. Configure Frontend Environment

Create `frontend/.env.local`:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_KEY=your-anon-public-key
```

Rebuild:
```bash
cd frontend
npm run build
```

## Part 3: Vercel Deployment (Frontend Only)

> **Important:** Vercel hosts the static frontend only. It cannot run the Node.js WebSocket backend. Users must connect the Vercel app to their local backend server.

### Automatic Vercel Setup

The project includes a `vercel.json` and root `package.json` that handle the build automatically:

```json
// vercel.json
{
  "outputDirectory": "frontend/dist"
}
```

```json
// package.json (root)
{
  "scripts": {
    "install": "cd frontend && npm install",
    "build": "cd frontend && npm run build"
  }
}
```

### Steps

1. Go to https://vercel.com/
2. Click **Add New Project**
3. Import your GitHub repo (`locomm`)
4. Vercel will auto-detect the root `package.json`
5. Click **Deploy**

### Environment Variables on Vercel

In Vercel project settings → **Environment Variables**, add:

| Name | Value |
|------|-------|
| `VITE_SUPABASE_URL` | Your Supabase project URL |
| `VITE_SUPABASE_KEY` | Your Supabase anon key |

> Note: The `VERCEL=1` environment variable is automatically set by Vercel. The `vite.config.js` uses this to set `base: "/"` automatically during Vercel builds.

## Part 4: Backend Hosting (Optional)

If you want users to access LoComm without running the server on a local machine, you can host the backend on a cloud provider.

### Option A: Render.com (Free Tier)

1. Go to https://render.com/
2. New → **Web Service**
3. Connect GitHub repo
4. **Root Directory**: `backend`
5. **Build Command**: `npm install`
6. **Start Command**: `node server.js`
7. Deploy

> ⚠️ Render's free tier sleeps after inactivity. Use a paid plan for production.

### Option B: Railway.app

1. Go to https://railway.app/
2. New Project → Deploy from GitHub
3. Select your repo
4. Set root directory to `backend`
5. Railway auto-detects `npm start`

### Option C: VPS / Cloud VM

1. SSH into your server
2. Install Node.js 18+
3. Clone the repo
4. Build frontend and start backend
5. Use `pm2` for process management

```bash
npm install -g pm2
cd backend && npm install
cd ../frontend && npm install && npm run build
cd ../backend && pm2 start server.js --name locomm
```

## Part 5: Creating GitHub Releases

### 1. Tag and Create Release

```bash
git tag v1.0.0
git push origin v1.0.0
```

Then on GitHub:
1. Go to **Releases → Create a new release**
2. Tag: `v1.0.0`
3. Title: `v1.0.0 — Initial Release`
4. Add release notes
5. Publish

### 2. Notify Users via Supabase

```sql
INSERT INTO updates (app_name, platform, version, download_url, release_notes)
VALUES (
  'locomm',
  'web',
  '1.0.1',
  'https://github.com/YOUR_USERNAME/locomm/releases/tag/v1.0.1',
  'Bug fixes and performance improvements'
);
```

The in-app update banner will appear automatically on the next load.
